---
title: "EZAnim:unragdoll() — Restore a Ragdolled Character"
sidebarTitle: "unragdoll()"
description: "Re-enable Motor6D joints, remove ragdoll physics constraints, and return the Humanoid to a standing state after a prior ragdoll() call."
---

`unragdoll()` reverses everything that [`ragdoll()`](/api/ragdoll) did to a model. It restores every `Motor6D` to the orientation it had before ragdolling, re-enables those joints so the animator regains control of the rig, and then destroys all `BallSocketConstraint` and `Attachment` instances that were added during ragdolling. Finally, it restores the `Humanoid` to a functional standing state by setting `PlatformStand` to `false`, `AutoRotate` to `true`, and transitioning the `Humanoid` into the `GettingUp` state so the character smoothly returns to locomotion. If `ragdoll()` was never called on the given model, `unragdoll()` does nothing.

## Parameters

<ParamField path="Model" type="Model" required>
  The ragdolled model to restore. This must be the same model reference that was
  passed to `ragdoll()`. If no ragdoll data exists for this model, the call is a
  safe no-op.
</ParamField>

## What Gets Restored

When `unragdoll()` runs, it performs the following steps in order:

1. **Motor6D joints** — Each joint's original transform, recorded before it was disabled by `ragdoll()`, is restored and the motor is re-enabled so the animator regains full control.
2. **Physics constraints** — Every `BallSocketConstraint` and `Attachment` that was created by `ragdoll()` is destroyed, removing all collision geometry added for the ragdoll simulation.
3. **Humanoid state** — `Humanoid.PlatformStand` is set to `false`, `Humanoid.AutoRotate` is set to `true`, and `Humanoid:ChangeState(Enum.HumanoidStateType.GettingUp)` is called so the character stands back up naturally.
4. **State cleanup** — All ragdoll tracking for this model is cleared, so subsequent calls to `unragdoll()` on the same model become safe no-ops.

## Notes

- Calling `unragdoll()` on a model that was never ragdolled is safe — the function checks for stored data first and exits early if none is found.
- After `unragdoll()` completes you can call `:play()` normally to resume animations; the `Humanoid` state transition to `GettingUp` handles the stand-up pose automatically.
- If you passed a `Duration` to `ragdoll()`, `unragdoll()` is called automatically and you do not need to call it again.

## Example

The example below implements a revival system: a ragdolled character is restored when another player interacts with a `ProximityPrompt`. After unragdolling, an idle animation is played to return the character to a neutral stance.

```lua Revive a Ragdolled Character
local EZAnim = require(game.ReplicatedStorage.EZAnim)

EZAnim:setup({
    Idle = "rbxassetid://123456789",
})

local character       = workspace.MyCharacter
local revivePrompt    = workspace.RevivePrompt -- a ProximityPrompt instance

revivePrompt.Triggered:Connect(function(playerWhoTriggered)
    print(playerWhoTriggered.Name, "is reviving", character.Name)

    -- Restore joints, remove constraints, stand the Humanoid back up
    EZAnim:unragdoll(character)

    -- Resume a neutral idle animation now that the rig is back under animator control
    EZAnim:play(character, "Idle")

    print(character.Name, "has been revived.")
end)
```
