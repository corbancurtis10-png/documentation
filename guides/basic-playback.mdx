EZAnim gives you a straightforward API for loading and controlling animations on any model that has a `Humanoid` or `AnimationController`. After a one-time call to `EZAnim.init()` you can play, stop, loop, and adjust any animation in your list without ever touching the Roblox `AnimationTrack` API directly.

## Initializing EZAnim

Before you can play anything you need to call `EZAnim.init()` and pass it your animation list along with any global defaults.

```lua ServerScript/AnimationManager.lua
local EZAnim = require(game.ReplicatedStorage.EZAnim)

local Anim = EZAnim.init({
    List = {
        Idle  = "rbxassetid://123456789",
        Walk  = "rbxassetid://234567891",
        Run   = "rbxassetid://345678912",
        Punch = "rbxassetid://456789123",
        Hit   = "rbxassetid://567891234",
    },
    DefaultFade   = 0.15,
    DefaultSpeed  = 1,
    DefaultWeight = 1,
    AutoCleanUp   = true,
})
```

`DefaultFade`, `DefaultSpeed`, `DefaultWeight`, and `AutoCleanUp` are all optional. EZAnim falls back to `0.15`, `1`, `1`, and `true` respectively when you omit them.

## Playing Animations

### Looped idle

Pass `Looped = true` to keep an animation cycling until you explicitly stop it.

```lua ServerScript/AnimationManager.lua
-- Play a looped idle animation at default settings
Anim:play(character, "Idle", "Idle", true)
```

### One-shot action with custom parameters

You can override fade time, speed, weight, and layer on a per-call basis. Any parameter you leave as `nil` falls back to the global defaults you set in `init`.

```lua ServerScript/AnimationManager.lua
-- Play a punch animation once: faster, full weight, on layer 1
Anim:play(
    character,   -- Model
    "Punch",     -- Id (key from your List)
    "Action",    -- Priority
    false,       -- Looped
    0.1,         -- Fade in
    1.2,         -- Speed (20 % faster than normal)
    1,           -- Weight
    1,           -- Layer
    "Combat"     -- Group tag (optional)
)
```

The `Group` tag is a free-form string you can use to categorise tracks. EZAnim does not enforce any behaviour on it â€” it is there for your own bookkeeping.

<Tip>
  EZAnim caches the underlying `Animation` instance the first time you play a given ID. Calling `:play()` a second time reuses that cached instance, so you never pay the cost of double-loading the same animation asset mid-session.
</Tip>

## Playing an Animation from the Beginning with `playOnce`

`:playOnce()` accepts exactly the same parameters as `:play()` but resets the track's `TimePosition` to `0` before starting playback. This makes it ideal for reactions where you want the animation to always snap to the first frame â€” even if the track was mid-way through from a previous play.

```lua ServerScript/AnimationManager.lua
-- Always restart the hit-react from frame 0
Anim:playOnce(
    character,
    "Hit",
    "Action",
    false,
    0.05
)
```

Use `:play()` when continuity matters (e.g. blending back into a looping walk) and `:playOnce()` when you need a hard restart (e.g. hit reactions, attack combos, emotes).

## Stopping Animations

### Stop a single animation

```lua ServerScript/AnimationManager.lua
-- Stop the Idle track with the default fade time
Anim:stop(character, "Idle")

-- Stop the Walk track with a longer custom fade
Anim:stop(character, "Walk", 0.4)
```

### Stop every animation on a model

```lua ServerScript/AnimationManager.lua
-- Immediately cut all animations (no fade)
Anim:stopAll(character, 0)

-- Fade everything out over 0.3 s
Anim:stopAll(character, 0.3)
```

`:stopAll()` is useful when you need to wipe the slate clean â€” for example, when a character dies or transitions into a cutscene.

## Quick Reference: Fine-Tuning a Playing Track

Once a track is playing you can adjust it without stopping and restarting it.

### `setSpeed`

```lua ServerScript/AnimationManager.lua
-- Slow the Run animation to half speed
Anim:setSpeed(character, "Run", 0.5)
```

### `setWeight`

```lua ServerScript/AnimationManager.lua
-- Blend the Idle animation to 50 % influence over 0.2 s
Anim:setWeight(character, "Idle", 0.5, 0.2)
```

The optional `Fade` parameter lets the weight change transition smoothly instead of snapping.

### `setTime`

```lua ServerScript/AnimationManager.lua
-- Jump to the halfway point of the Punch animation
Anim:setTime(character, "Punch", 0.25)
```

`setTime` is handy for syncing animations to game events â€” for example, locking a sword swing to the exact frame where the hitbox should activate.
